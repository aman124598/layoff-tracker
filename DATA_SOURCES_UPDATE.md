# Multi-Source Layoff Data Enhancement

## Overview
The layoff tracker application now fetches layoff data from multiple verified sources, ensuring comprehensive and up-to-date information. This document outlines the new features and changes.

## Data Sources

The application now aggregates data from:

1. **NewsAPI** (`newsapi.org`)
   - Real-time news articles about company layoffs
   - Global coverage with focus on major tech companies
   - Verified through keyword and company name validation

2. **Layoffs.fyi** (GitHub Dataset)
   - Curated layoff dataset with detailed information
   - One of the most reliable and widely-referenced layoff databases
   - Historical data from 2020 onwards
   - Source: https://github.com/MohammedTaherMcMoran/Layoffs.fyi-Dataset

3. **Alternative Financial News Sources**
   - Bloomberg, CNBC, Reuters, Financial Times, TechCrunch
   - Financial news focused on workforce reductions
   - Integrated through NewsAPI

## Key Features

### 1. Data Validation
- **Company Verification**: Only known major companies are tracked
- **Confirmed Layoffs**: Only past-tense confirmed layoffs (not rumors or planned)
- **Employee Count Validation**: Realistic numbers (50-99,999 range)
- **Deduplication**: Prevents duplicate entries from multiple sources

### 2. Source Tracking
- Each record now includes the source name and URL
- Database schema updated with `source_name` field
- Users can filter data by source

### 3. Backend Endpoints

#### Fetch All Layoffs
```
GET /api/layoffs
```
Returns all deduplicated layoff records with source information.

#### Get Available Sources
```
GET /api/layoffs/sources
```
Returns list of all data sources with record counts:
```json
{
  "sources": [
    { "name": "Layoffs.fyi", "count": 150 },
    { "name": "NewsAPI", "count": 89 },
    { "name": "Bloomberg", "count": 12 }
  ],
  "total": 251
}
```

#### Filter by Source
```
GET /api/layoffs/by-source?source=Layoffs.fyi
```
Returns all layoff records from a specific source.

#### Source Statistics
```
GET /api/layoffs/sources/stats
```
Detailed statistics about sources:
```json
{
  "total_records": 251,
  "sources": {
    "Layoffs.fyi": {
      "count": 150,
      "employees_total": 45000,
      "first_seen": "2024-01-15T00:00:00Z",
      "last_seen": "2024-04-18T00:00:00Z"
    }
  },
  "by_country": {
    "India": 85,
    "USA": 156,
    "UK": 10
  },
  "last_updated": "2024-04-18T12:30:00Z"
}
```

#### Sync Data from All Sources
```
POST /api/layoffs/sync
```
Manually trigger a sync from all sources. Returns:
```json
{
  "message": "Sync complete",
  "saved": 25,
  "skipped": 15,
  "total_fetched": 40,
  "sources": {
    "newsapi": 40,
    "layoffsfyi": 0,
    "alternative": 0
  }
}
```

## Database Schema Changes

### New Table Schema
```sql
CREATE TABLE IF NOT EXISTS layoffs (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    company_name TEXT NOT NULL,
    layoff_date DATE NOT NULL,
    employees_laid_off INTEGER,
    country TEXT NOT NULL,
    industry TEXT NOT NULL,
    source_url TEXT NOT NULL UNIQUE,
    source_name TEXT DEFAULT 'Unknown' NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL
);
```

### New Index
```sql
CREATE INDEX IF NOT EXISTS idx_layoffs_source ON layoffs(source_name);
```

```env
NEWS_API_KEY=your_newsapi_key_here

SUPABASE_ANON_KEY=your_supabase_anon_key
PORT=5000
```

Get a free NewsAPI key from: https://newsapi.org/

- **Initial Delay**: 5 seconds after server startup
- **Sources**: All three sources checked in parallel for optimal performance

## Data Quality Assurance

1. **Company Validation**: Only predefined major companies
2. **Deduplication**: By URL and company+count combination
3. **Date Validation**: Extracted from article dates
4. **Employee Count Ranges**: 50-99,999 (unrealistic numbers rejected)
5. **Layoff Status**: Must be confirmed (past tense), not planned or rumored
6. **Geographic Tracking**: Categorized by country with India-specific focus

## Troubleshooting

### No Data Appears After Sync
1. Check NewsAPI key is valid: `echo $NEWS_API_KEY`
2. Verify Supabase connection
3. Check server logs for errors
4. Ensure database schema is updated

### Duplicate Entries
1. Run cleanup endpoint: `POST /api/layoffs/cleanup`
2. Remove unrealistic entries: `POST /api/layoffs/cleanup-large`

### Source Information Missing
1. Ensure database migration is applied
2. Resync data: `POST /api/layoffs/sync`
3. Check backend logs for validation errors
